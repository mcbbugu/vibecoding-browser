# VibeCoding DevDock - Landing Page Spec

> **AI 落地页生成指南：** 这是一个为前端开发者打造的桌面应用，核心价值是统一管理所有本地开发项目，消除端口记忆负担。目标用户是同时维护多个项目的开发者。

---

## 🎯 产品定位 (Hero Section)

**主标题：** 别再记端口号了  
**副标题：** 一个地方管理所有本地开发项目，拖拽置顶、实时预览、秒速切换

**核心痛点：**
- 同时开发 3 个项目，来回切浏览器 Tab，localhost:3000 还是 5173？
- 想看移动端效果，得打开 DevTools，还得手动刷新
- 项目路径散落各处，找个文件要在 Finder 里翻半天

**解决方案一句话：**  
把所有本地服务变成可视化卡片，拖拽排序、点击直达、实时预览，像管理 App 一样管理你的开发项目。

---

## ✨ 核心功能 (Features Section)

### 1. 智能发现本地服务
**场景：** 启动了新项目但忘了端口号  
**功能：**
- 自动扫描 3000-10000 端口
- 快速扫描常用端口 (3000/5173/8080/5000/4200/8888)
- 实时显示运行状态（绿点 = 在线）
- 支持手动添加远程项目（staging/production）

**视觉建议：** 展示扫描动画 + 服务卡片逐个出现的效果

---

### 2. 实时预览 + 响应式调试
**场景：** 需要同时查看桌面端和移动端效果  
**功能：**
- 卡片内嵌 iframe 实时预览（无需打开浏览器）
- 一键切换设备尺寸（iPhone/iPad/Desktop）
- 内置 DevTools，直接调试
- 支持截图保存

**视觉建议：** 并排展示同一项目的手机/桌面预览效果对比

---

### 3. 拖拽置顶 + 个性化排序
**场景：** 核心项目要快速访问，临时项目放后面  
**功能：**
- 拖拽卡片到「置顶区」固定
- 自由调整卡片顺序
- 分区管理：置顶 vs 自动发现
- 自动保存布局

**视觉建议：** GIF 动画演示拖拽过程

---

### 4. 编辑器无缝集成
**场景：** 看到 Bug 想立刻改代码  
**功能：**
- 右键菜单 → 在 VSCode/Cursor/Windsurf 打开
- 一键打开项目目录（Finder/Terminal）
- 支持自定义编辑器路径

**视觉建议：** 展示右键菜单 + 编辑器自动打开的流程

---

### 5. 多项目 Tab 管理
**场景：** 需要在 3 个项目间频繁切换  
**功能：**
- 浏览器式 Tab 标签页
- `Cmd+T` 快速搜索项目
- `Cmd+W` 关闭当前 Tab
- 支持键盘快捷键（前进/后退/刷新）

**视觉建议：** 类似 Chrome 的 Tab 栏 + 搜索框高亮

## 🎨 界面设计参考 (UI/UX Guidelines)

**整体风格：**
- 现代极简，参考 Notion/Linear 的设计语言
- 深色/浅色主题切换
- 卡片式布局，圆角 12px，悬浮阴影
- 动画流畅（拖拽/切换都有过渡效果）

**颜色方案：**
- 主色调：Indigo 500（操作按钮）
- 状态色：Emerald 500（运行中）/ Zinc 400（已停止）
- 背景：浅色模式 Zinc 50 / 深色模式 #111111

**核心组件：**
1. **项目卡片：**
   - 顶部：iframe 预览区（16:9）
   - 底部：项目名称 + 运行状态小圆点 + 路径/URL
   - 悬浮态：显示阴影 + 轻微上浮

2. **置顶区：**
   - 固定高度（445px），响应式网格布局
   - 空态：虚线边框 + 提示文案"拖拽项目到这里置顶"

3. **发现区：**
   - 横向滚动（<2XL 屏幕）或网格布局（≥2XL）
   - 自动刷新按钮（右键显示扫描模式菜单）

## 🚀 使用流程 (How It Works)

**步骤 1：** 启动应用，自动扫描本地运行的服务  
**步骤 2：** 点击卡片直接打开项目，或拖拽到置顶区固定  
**步骤 3：** 右键卡片选择"在编辑器中打开"或"查看文件夹"  
**步骤 4：** 使用 Cmd+T 搜索项目，快速跳转
