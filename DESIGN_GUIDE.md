# DevDock 落地页设计指南

## 产品定位
**AI 原生开发者的本地项目浏览中心**
- 通过**智能端口扫描**自动发现本地运行的项目
- 提供**内置浏览器预览**，告别浏览器标签混乱
- 为多项目并行开发场景优化
- 目标用户：Cursor/Windsurf 重度使用者，同时维护多个项目

## 核心痛点
1. **浏览器标签地狱** - 同时开发多个项目时，Chrome 里 10+ 个 localhost 标签，找不到对应哪个项目
2. **端口记忆负担** - 项目跑起来了，但忘记端口号，要去终端翻日志或一个个试
3. **编辑器窗口混乱** - 同时打开多个编辑器窗口（Cursor/VS Code），Cmd+Tab 切换找不到对应项目
4. **响应式测试麻烦** - 测试不同设备尺寸要开一堆浏览器窗口或用 DevTools 手动调整
5. **项目状态不可见** - 不知道哪些项目在运行，哪些停了，哪些报错了

## 核心功能
### 1. 智能端口扫描（最核心）
- **自动发现**本地运行的项目，扫描常用端口
- 实时显示项目端口号、运行状态
- 一键访问项目预览

### 2. 内置浏览器预览
- 项目预览直接在应用内，**不用开一堆 Chrome 标签**
- **18+ 设备预设**：iPhone、iPad、Desktop 等，一键切换
- 完整 Chromium DevTools，开发调试方便
- `Cmd+Shift+S` 截图到剪贴板

### 3. 可视化项目管理
- 卡片展示所有项目，运行状态一目了然
- **拖拽排序**、固定常用项目
- 记住每个项目的 URL 和设备偏好
- 右键菜单：在编辑器打开、在访达中打开

### 4. 一键跳转编辑器（高效核心）
- `Cmd+E` 或右键菜单，直接打开对应项目的编辑器
- 不用在 10 个 Cursor 窗口里找项目，**精准跳转**
- 支持 Cursor、VS Code、Windsurf、WebStorm 等主流编辑器
- 其他快捷键：`Cmd+T` 搜索项目、`Cmd+L` 地址栏、`Cmd+[/]` 前进后退

### 5. 项目上下文记忆
- 记住每个项目的 URL、设备尺寸、编辑器配置
- 下次打开项目，环境自动恢复
- 真正的"一站式开发中心"

## 设计理念
**极简 · 高效 · 开发者友好**

### 视觉风格
- **深色优先**：长时间使用不累眼，符合开发者习惯
- **信息密度适中**：重要信息突出，次要信息收起
- **微动效**：只在关键交互处使用，不花哨
- **现代简约**：圆角、毛玻璃、渐变适度使用

### 交互原则
- **零学习成本**：符合 macOS 操作习惯
- **快捷键覆盖所有核心功能**
- **拖拽即可完成大部分操作**
- **状态清晰可见**（运行中/停止/错误）

## 配色方案
基于应用实际使用的颜色：

### 主色调
- **强调色**：Indigo-500 `#6366F1`（紫蓝色）
  - 用于：按钮、选中状态、焦点框、加载动画
- **强调色-深**：Indigo-600 `#4F46E5`
  - 用于：按钮 hover 状态

### 背景
- **亮色模式**：Zinc-50 `#FAFAFA`
- **暗色模式**：深灰 `#111111`

### 中性色
- **卡片/容器**：
  - 亮色：White + Zinc-200 边框
  - 暗色：Zinc-800 + Zinc-700 边框
- **文字**：
  - 主要文字：Zinc-900 / Zinc-100
  - 次要文字：Zinc-600 / Zinc-300
  - 辅助文字：Zinc-500 / Zinc-400

### 状态色
- **成功/运行中**：Green-500 `#22C55E`
- **警告**：Yellow-500 `#EAB308`
- **错误**：Red-500 `#EF4444`
- **信息**：Indigo-500 `#6366F1`

## Logo 使用
- **文件**：`logo-dark.svg`（暗色背景用）、`logo-light.svg`（亮色背景用）
- **设计**：黑板 + 终端图标，白色/黑色填充
- **尺寸**：导航栏 24px，页面标题处可适当放大

## 参考竞品（差异化点）
- **vs LocalCan/Valet**：不只是端口管理，更关注开发工作流
- **vs Herd/Laragon**：不绑定特定语言，支持所有 Node.js 项目
- **vs Proxyman**：不是网络调试工具，是项目管理中心
- **vs Arc Browser**：不是浏览器，是带预览的项目管理器

## 落地页结构建议
1. **Hero 区**：Slogan + 核心价值主张 + 下载按钮 + 产品截图/演示
2. **痛点区**：3-4 个开发者常见痛点，配简洁插图
3. **功能区**：5 个核心功能，每个一个卡片，配动态演示或 GIF
4. **技术栈**：横向滚动展示，突出现代化技术
5. **快捷键展示**：交互式演示，突出"键盘优先"理念
6. **CTA 区**：再次强调下载，加上 GitHub Star 按钮
7. **Footer**：社交链接、文档、隐私政策

## 设计要点
- **移动端友好**：虽然产品只支持 macOS，但落地页要响应式
- **加载速度**：避免大图，优先 SVG/WebP
- **SEO 友好**：语义化标签，meta 信息完整
- **暗色优先**：默认暗色主题，可切换
- **动画克制**：只在 Hero 区和功能演示处用动画
